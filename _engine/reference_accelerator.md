# Reference Accelerator — v4.2

Determine **reference class** quickly after capturing idea nucleus.

---

## Classification

| Level | Signals | Conversation shift |
| ----- | --------- | -------------------- |
| full_clone | “same as X”, “clone of” | preload entire pattern; confirmations |
| partial_clone | “like X but for Y”, “Airbnb style” | preload intersection; deepest diff probing |
| inspired_by | borrow UX wording | metaphors confirmed; substantive discovery persists |
| greenfield | no anchor | exhaustive modules |

Multi-reference acceptable (up to three). Maintain ordered influence list (`primary_reference`, secondary).

---

## Diff-check template

Assistant states assumed baseline bullets (flows, monetization guesses, surfaces). User marks ✅ / ⚠ tweak / ❌ irrelevant.

Questions must target deltas only unless gap risk flagged.

---

## Skip / probe rules

Skip generic marketplace boilerplate discovery when preload covers it; **still** run constrained modules that encode user deviations (payments region, moderation twist).

Inspired-by avoids assuming domain data model.

---

## Anti-hallucination

When uncertain about reference internals, downgrade claim language (“Commonly…” + confirm).

---

---

### Multi-reference handling

Prefer unified merge where non-conflicting. Surface explicit fork when references imply incompatible architectures (e-commerce vs realtime game).
